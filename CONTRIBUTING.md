# Contributing

Thanks for helping improve timestable.

## Workflow

1. Create a branch for your change.
2. Keep the change focused on one behavior or bug.
3. Run the relevant frontend or backend checks before opening a pull request (completely manual right now)
4. Describe the user-visible impact in the pull request.
5. Deployment onto prod server is done through infrastructure independent of Github by @rempferg (to be automated with Github actions at some point)

## Code quality

- Follow the existing Angular and Python style in the repo.
- Do not commit secrets, environment files, or local build artifacts.
